## 9. References & Resources

### Books (Priority Order for This Goal)

| # | Book | Author | Why |
|---|---|---|---|
| 1 | **Design Patterns: Elements of Reusable OO Software** | Gang of Four | Canonical reference; read Chapters 1-2, then by pattern as needed |
| 2 | **Effective Java (3rd ed.)** | Joshua Bloch | Java-specific patterns and idioms; Builder, Singleton, Factory, Flyweight |
| 3 | **Refactoring (2nd ed.)** | Martin Fowler | Patterns in action — before/after; links code smells to fixes |
| 4 | **Clean Code** | Robert C. Martin | SOLID in practice; code-level design |
| 5 | **Head First Design Patterns (2nd ed.)** | Freeman & Robson | Most approachable for building intuition before GoF |

| Resource | What You'll Find |

### Practice

- **GitHub: iluwatar/java-design-patterns** — clone and study the test cases; they show intent clearly
- For each pattern, implement it from scratch, then look at the reference implementation
- Code review exercise: find pattern violations in open-source Spring projects
